# 智慧路灯系统 - 需求分析文档

## 1. 项目概述

### 1.1 项目背景
智慧路灯系统是重庆交通大学-中软国际集中实训项目之一，旨在通过「端-智-云一体化」架构，实现路灯的智能控制和节能管理。

### 1.2 项目目标
构建一套智慧路灯物联网系统，实现：
- 实时监测环境光照强度
- 根据光照阈值自动开关路灯
- 支持手动远程控制路灯
- 设备状态监控与离线告警
- 历史数据可视化展示
- 智能问答（AI辅助）

### 1.3 技术路线
- **端侧（校内模拟）**：Mock数据发生器模拟光照传感器
- **端侧（基地真实）**：鸿蒙开发板(Hi3861/Hi3516) + 光照传感器 + 继电器
- **通信**：MQTT协议
- **后端**：Java Spring Boot 3.x
- **前端**：Vue 3 + Element Plus + ECharts
- **数据库**：MySQL

| 角色 | 说明 | 主要需求 |

### 模块一：数据监测

#### F1.1 光照强度实时监测

**页面/组件：** 实时监测页（MonitorView.vue）

**数据流：**
- 页面加载时调用 `GET /devices/{deviceId}/sensor-data/latest` 获取当前光照值
- 建立 WebSocket 连接 `ws://localhost:8080/ws/monitor`，监听 `type: "SENSOR_DATA"` 消息实时更新
- 使用 Pinia store（`useMonitorStore`）管理当前光照数据和设备列表

**UI布局：**
- 顶部为设备选择器（下拉框/卡片式切换），展示所有路灯设备
- 主体区域显示当前选中设备的**光照强度数值**（大号字体数字 + Lux单位）
- 配合环形进度条或色条（0-2000Lux范围，低亮度用深色、高亮度用亮色渐变）
- 附加展示：最后上报时间、设备当前开关状态

**状态处理：**
| 状态 | 表现 |
|------|------|
| 加载中 | Skeleton骨架屏，数值区域显示脉冲占位 |
| WebSocket断开 | 显示"连接中…"提示条 + 轮询回退（每5秒拉取latest接口） |
| 无数据 | 显示 "-- Lux" + "等待设备上报…" 空状态插图 |
| 数值异常（负值/超2000） | 显示"数据异常"警告徽标，保留最后正常值 |
| 设备离线 | 数值区域半透明叠加"设备已离线"遮罩层 |

**交互：**
- 设备切换时，数值有过渡动画（旧值淡出、新值淡入）
- 鼠标悬停显示该设备最近10条数据的小型趋势微缩图
- 数值变化幅度 > 100Lux 时触发轻微的数值闪烁提示

#### F1.2 设备状态监控

**页面/组件：** 复用实时监测页 / 独立卡片组件（DeviceStatusCard.vue）

**数据流：**
- 页面加载时调用 `GET /devices` 获取所有设备
- WebSocket 监听 `type: "DEVICE_STATUS"` 消息，实时更新设备在线/离线状态和灯光状态
- 心跳超时由后端判断（30秒无心跳 → offline），前端只需展示结果

**UI布局：**
- **设备状态总览条**：顶部显示汇总统计「总设备 N | 在线 N | 离线 N | 亮灯 N」
- **设备卡片网格**（每行4列响应式），每张卡片包含：
  - 设备名称 + 位置（小字）
  - 状态指示灯（绿色在线 / 灰色离线 / 红色告警）
  - 灯光状态图标（💡亮/暗）
  - 控制模式标签（自动/手动）
  - 最后心跳时间（"3秒前"相对时间）
- 卡片按状态分组：在线在前、离线在后，离线卡片置灰

**状态处理：**
| 状态 | 表现 |
|------|------|
| 加载中 | 卡片区域显示 6× Skeleton卡片（灰色圆角方块） |
| 接口报错 | 卡片区域中央显示"加载失败，点击重试"按钮 |
| 设备列表为空 | 显示空状态插图 + "暂无设备，请先添加设备" + 「去添加」按钮跳转设备管理 |
| 状态切换动画 | 设备从在线→离线时卡片闪烁红色边框1秒后置灰 |
| WebSocket重连 | 底部显示"实时连接已恢复" Toast 提示 |

**交互：**
- 点击卡片 → 跳转到该设备的详情/控制页
- 卡片支持按状态筛选（Tab切换：全部 / 在线 / 离线 / 告警）
- 支持搜索框按设备名称/位置模糊搜索

### 模块二：数据可视化

#### F2.1 历史光照趋势图

**页面/组件：** 历史趋势页（HistoryView.vue） / ECharts折线图组件

**数据流：**
- 调用 `GET /devices/{deviceId}/sensor-data?start=...&end=...` 获取历史数据
- 调用 `GET /devices/{deviceId}/sensor-data/stats?start=...&end=...` 获取统计摘要

**UI布局：**
- **顶部筛选栏**：
  - 设备选择下拉框
  - 时间范围选择器：预设快捷选项（最近1小时 / 6小时 / 24小时 / 7天 / 自定义）
  - 粒度切换按钮（原始数据 / 5分钟聚合 / 1小时聚合）
- **统计摘要卡片**（横向排列4个指标）：平均值 / 最大值 / 最小值 / 数据点数
- **ECharts折线图**（占页面主体70%）：
  - X轴：时间（根据粒度显示格式：HH:mm / HH:mm / MM-dd）
  - Y轴：光照强度（Lux），范围0-2000
  - 两条参考线：开灯阈值线（红色虚线50Lux）和关灯阈值线（绿色虚线100Lux）
  - 数据点 hover 显示 Tooltip：时间 + 精确数值
  - 面积填充（半透明渐变）：阈值线以下区域浅灰色填充表示"应开灯时段"
- **数据表格**（图表下方可折叠）：分页展示原始数据列表

**状态处理：**
| 状态 | 表现 |
|------|------|
| 加载中 | 图表区域显示 ECharts loading 动画 |
| 选中时间段无数据 | 图表显示 "暂无数据" 文字浮层 + 提示"请选择其他时间段" |
| 数据点过密（>1000条） | 前端自动降采样（LTTB算法），并在图例标注"已降采样（N条→M条）" |
| 接口异常 | 图表区域显示"数据加载失败" + 重试按钮 |
| 设备切换 | 图表清空 → loading → 新数据入场动画 |

**交互：**
- 图表支持 ECharts 内置的缩放（dataZoom）、平移、导出为图片
- 点击图表上的数据点 → 显示该时刻的详细传感器数据
- 图例可点击切换显示/隐藏阈值参考线
- 时间范围选择后自动触发数据请求

#### F2.2 设备总览 Dashboard

**页面/组件：** Dashboard页（DashboardView.vue）

**数据流：**
- 调用 `GET /dashboard/stats` 获取总览统计数据
- 调用 `GET /devices` 获取设备列表
- WebSocket 监听设备状态变更实时刷新

**UI布局（大屏风格，栅格布局）：**
- **第一行（统计卡片）**：4张统计卡片并排
  - 总设备数（📦 图标，蓝色主题）
  - 在线设备数（🟢 图标，绿色主题）
  - 离线设备数（🔴 图标，红色主题，数字闪烁如果 > 0）
  - 待处理告警数（⚠️ 图标，橙色主题）
- **第二行（左右两栏）**：
  - 左栏：**设备在线率环形图**（ECharts 环形图，绿色/灰色占比，中央显示"在线率 80%"）
  - 右栏：**路灯开关状态分布**（ECharts 饼图，亮灯/关灯数量及占比）
- **第三行（全宽）**：**各设备光照强度实时折线图**（多系列折线图，每设备一条线，仅展示最近1小时数据）
- **第四行**：**最近告警列表**（紧凑表格，仅显示最近5条未处理告警）+「查看全部」链接

**状态处理：**
| 状态 | 表现 |
|------|------|
| 加载中 | 每个卡片/图表区域独立显示 Skeleton 或 loading |
| 部分数据加载失败 | 失败的模块显示"加载失败"占位，不影响其他模块 |
| 全部数据加载失败 | 页面中央显示完整错误页 + 「重新加载」按钮 |
| 空数据 | 统计卡片显示 0，图表显示"暂无数据"空状态 |
| WebSocket实时更新 | 卡片数值和图表有过渡动画更新，不闪烁整个页面 |

**交互：**
- 点击统计卡片 → 跳转到对应详情页（如点击离线数 → 跳转设备管理并筛选离线状态）
- 点击告警条目 → 跳转告警管理页
- 支持 F5 手动刷新，也支持页面可见性变化时自动刷新
- 图表 hover 显示数值 tooltip

### 模块三：设备控制

#### F3.1 路灯光照联动（自动控制）

**页面/组件：** 设备控制页中的自动模式状态展示（ControlView.vue）

> 联动逻辑在后端实现，前端只需展示当前控制模式和联动状态

**数据流：**
- 调用 `GET /devices/{id}` 获取设备详情（含 `controlMode`、`lightStatus`、`thresholdOn`、`thresholdOff`）
- WebSocket 监听 `type: "DEVICE_STATUS"` 获取灯光状态变更
- WebSocket 监听 `type: "CONTROL_RESULT"` 获取自动控制执行结果

**UI布局：**
- **模式状态指示器**：当前控制模式标签（自动模式 🟢 / 手动模式 ⚪）
- **联动状态卡片**：
  - 当前光照值（实时更新）
  - 开灯阈值线 / 关灯阈值线（与光照值对比的可视化标尺）
  - 自动联动日志：最近5条自动开关记录（时间 + 触发原因）
- **模式切换开关**（Switch 组件，支持自动/手动切换）

**状态处理：**
| 状态 | 表现 |
|------|------|
| 自动模式 | 显示绿色"自动控制中"徽标，联动状态卡片完整展示 |
| 手动模式 | 显示灰色"手动控制中"徽标 + 提示"自动联动已暂停" |
| 控制指令下发中 | 按钮显示 loading，结果通过 WebSocket 返回后更新 |
| 控制失败 | 显示"控制失败"错误提示，灯光状态回滚 |

**交互：**
- 模式切换：Switch 开关，切换后调用 `PUT /devices/{id}/control-mode`
- 切换确认弹窗：从自动→手动时提示"切换手动模式后自动联动将暂停，是否继续？"

#### F3.2 路灯手动控制

**页面/组件：** 设备控制页（ControlView.vue） / 操作面板

**数据流：**
- 调用 `POST /devices/{deviceId}/control` 下发开/关指令
- 通过 WebSocket `type: "CONTROL_RESULT"` 接收执行结果
- 调用 `GET /devices/{deviceId}/sensor-data/latest` 获取当前光照值作为控制参考

**UI布局：**
- **设备选择区**：顶部设备卡片列表（展示设备名、当前位置状态、在线/离线）
  - 在线设备可操作，离线设备置灰并显示"设备离线，无法控制"
- **控制面板**（选中设备后展示）：
  - 大号灯光状态图标（💡 亮/灭动画效果）
  - 开关按钮组：开灯按钮（黄色高亮）/ 关灯按钮（灰色），当前状态高亮
  - 附加信息：当前光照值、控制模式、最后操作时间
- **控制日志**（下方表格）：最近20条手动控制记录（时间、指令、来源、结果）

**状态处理：**
| 状态 | 表现 |
|------|------|
| 未选择设备 | 面板显示"请选择一个设备"引导文字 |
| 设备离线 | 按钮禁用 + 提示"设备离线，无法控制" |
| 指令下发中 | 按钮进入 loading 状态，禁用重复点击 |
| 指令成功 | 灯光图标切换动画（亮/灭过渡），显示"操作成功" Toast |
| 指令失败 | 按钮恢复，显示红色"操作失败"提示 + 失败原因 |
| 快速连续点击 | 防抖处理（1秒内重复点击忽略） |
| 手动模式与自动模式切换 | 操作日志中标注 source 为 manual/auto |

**交互：**
- 开/关按钮使用防抖（debounce 500ms），防止重复提交
- 操作后通过 WebSocket 等待结果回执，超时（5秒）则提示"指令已下发，等待设备响应…"
- 支持键盘快捷键：选中设备后按 O 开灯、按 F 关灯

### 模块四：参数管理

#### F4.1 光照阈值设置

**页面/组件：** 参数设置页 / 弹窗表单（ThresholdDialog.vue / ThresholdSetting.vue）

**数据流：**
- 加载：调用 `GET /devices/{id}` 获取当前阈值（`thresholdOn`、`thresholdOff`）
- 保存：调用 `PUT /devices/{id}/threshold` 提交新阈值

**UI布局：**
- **阈值卡片**（每个设备独立设置）：
  - 开灯阈值（thresholdOn）：滑块组件 + 数字输入框，范围 0-500 Lux，步长 5
  - 关灯阈值（thresholdOff）：滑块组件 + 数字输入框，范围 0-500 Lux，步长 5
  - 视觉提示：标尺图上标记当前阈值位置，与实时光照值对比
- **批量设置**：支持"应用到所有设备"复选框
- **预设方案**：下拉选择预设（黄昏模式 50/100 / 阴天模式 80/150 / 夜间模式 20/50）

**状态处理：**
| 状态 | 表现 |
|------|------|
| 加载中 | 滑块和输入框显示 Skeleton |
| 保存中 | 保存按钮 loading，禁用表单 |
| 保存成功 | 显示"保存成功" + 自动关闭弹窗（Toast风格） |
| 校验失败 | 输入框下方红色提示文字 |
| 阈值逻辑校验 | thresholdOn 必须 < thresholdOff，否则按钮禁用 + 提示"开灯阈值必须小于关灯阈值" |

**交互：**
- 滑块拖动时数字输入框同步更新，输入框修改时滑块同步滑动
- 修改后未保存切换设备时弹出"是否保存更改"确认框
- 提供"恢复默认"按钮（50/100 Lux），点击后滑块和输入框归位

**校验规则：**
| 字段 | 规则 |
|------|------|
| thresholdOn | 0~500，整数或一位小数，必须 < thresholdOff |
| thresholdOff | 0~500，整数或一位小数，必须 > thresholdOn |

### 模块五：告警管理

#### F5.1 设备离线告警

**页面/组件：** 告警管理页（AlarmView.vue） / 告警通知组件

**数据流：**
- 页面加载时调用 `GET /alarms?status=pending&type=offline` 获取未处理告警
- WebSocket 监听 `type: "NEW_ALARM"` 实时接收新告警
- 处理告警：调用 `PUT /alarms/{id}/resolve`

**UI布局：**
- **告警横幅**（页面顶部，仅在有未处理告警时显示）：
  - 红色警告条 + 滚动显示最新告警摘要
  - 点击可快速定位到告警列表
- **告警列表**（表格形式）：
  | 列 | 说明 |
  |----|------|
  | 严重级别 | 彩色标签（critical 🔴 / warning 🟡 / info 🔵） |
  | 告警类型 | 中文标签（离线 / 传感器异常） |
  | 设备名称 | 可点击跳转设备详情 |
  | 告警内容 | 简要描述 |
  | 时间 | 相对时间 + 绝对时间（hover显示） |
  | 状态 | 待处理 / 已处理 标签 |
  | 操作 | 「处理」按钮（仅待处理状态显示） |
- **告警统计**：侧边统计卡片（今日告警数 / 未处理数 / 平均处理时长）

**状态处理：**
| 状态 | 表现 |
|------|------|
| 加载中 | 表格显示 Skeleton 行 |
| 无告警 | 表格显示空状态 "✅ 当前无告警，一切正常" 大号图标 |
| 新告警实时推送 | 列表顶部插入新行 + 高亮闪烁（黄色背景渐隐动画） |
| WebSocket断开 | 顶部提示"实时告警已暂停"，自动重连后恢复 |
| 处理告警中 | 操作按钮 loading，完成后该行置灰显示"已处理" |

**交互：**
- 新告警推送时，浏览器标签页标题闪烁显示"(N) 新告警"
- 告警支持批量处理（勾选多条 → 「批量处理」按钮）
- 列表支持筛选：按类型（全部/离线/传感器异常）、按状态（全部/待处理/已处理）
- 支持按时间排序（正序/倒序）

#### F7.1 维护智能问答

**页面/组件：** 智能问答页（ChatView.vue） / 聊天组件（ChatPanel.vue）

**数据流：**
- 调用后端智能体 API（由后端对接 MaxKB 平台 RAG 知识库）
- 请求格式（具体按后端定义）：
  ```json
  {

  {
    "answer": "可能原因：1. 接触不良 2. 传感器异常...",
    "sources": ["维护手册v2.3", "故障排查指南"],
    "suggestions": ["建议检查接线端子"]
  }
  ```

**UI布局：**
- **聊天面板**（左侧主区域，类似即时通讯）：
  - 消息列表：用户消息右对齐（蓝色气泡），AI回复左对齐（白色气泡带图标）
  - 输入框：底部固定，支持 Enter 发送、Shift+Enter 换行
  - 发送按钮
  - 清空对话按钮（右上角垃圾桶图标）
- **快捷问题面板**（右侧侧边栏，可折叠）：
  - 预设问题列表（点击自动填入发送）：
    - "如何判断传感器是否故障？"
    - "路灯离线了怎么办？"
    - "如何设置光照阈值？"
    - "设备控制没有响应怎么办？"
- **消息内容渲染**：支持 Markdown 格式（代码块、列表、加粗等）

**状态处理：**
| 状态 | 表现 |
|------|------|
| 首次进入 | 显示欢迎消息 + 3-4个快捷问题卡片 |
| 加载中（AI回复） | 气泡显示打字动画（...闪烁） |
| AI回复完成 | 打字动画停止，完整内容显示 |
| 接口超时/报错 | 气泡显示红色"回复失败" + 「重试」按钮 |
| 网络断开 | 输入框禁用 + 提示"网络连接已断开" |
| 空输入 | 发送按钮禁用 |
| 输入过长（>500字） | 显示字数统计 + 超出限制时禁用发送 |

**交互：**
- 消息列表自动滚动到底部（新消息到达时）
- 支持复制 AI 回复内容（hover 出现复制图标）
- 快捷问题点击后自动发送，同时右侧面板收起
- 输入框支持清除按钮（X 图标）
- 支持上下文记忆（当前会话内的连续对话）
- 消息历史保存在 Pinia store 中，刷新页面清空（不持久化）

**边缘情况：**
- 连续快速发送：只处理最后一条，前序请求取消（使用 AbortController）
- AI 回复过长：消息气泡最大高度 400px，超出显示「展开全文」按钮
- 敏感词拦截：前端简单过滤（提示"请重新描述问题"），后端做主要拦截

## 4. 业务流程

### 流程一：光照联动自动开关路灯
```
1. 市政人员设置光照阈值
2. 系统实时监测光照强度（Mock数据/真实传感器）
3. 判断光照是否低于阈值？
   ├─ 是 → 自动下发开灯指令 → 路灯开启 → 状态反馈
   └─ 否 → 持续监测
4. 光照高于阈值时自动关灯
```

### 流程二：设备离线告警处理
```
1. 设备定期发送心跳信号（MQTT）
2. 后端监测心跳
3. 心跳中断超过阈值时间？
   ├─ 是 → 触发离线告警 → 通知管理员
   └─ 否 → 正常记录
4. 管理员处理 → 设备恢复上线
```

## 5. 数据定义

### 5.1 光照强度
- **范围**：0 ~ 2000 Lux
- **阈值建议**：
  - 开灯阈值：< 50 Lux（黄昏/阴天）
  - 关灯阈值：> 100 Lux（天亮）

### 5.2 设备状态
- **online**：在线（心跳正常）
- **offline**：离线（心跳超时 > 30秒）

### 5.3 告警类型
- **offline**：设备离线告警
- **sensor_abnormal**：传感器数据异常
