# 智慧路灯系统 - 需求分析文档 (v5)

## 1. 项目概述

### 1.1 项目背景
智慧路灯系统是重庆交通大学-中软国际集中实训项目之一，旨在通过「端-智-云一体化」架构，实现路灯的智能控制和节能管理。

### 1.2 项目目标
构建一套智慧路灯物联网系统，实现：
- 多类型传感器实时监测（光照、温度、湿度、功率）
- 根据光照阈值自动开关路灯
- 支持手动远程控制路灯
- 设备状态监控与离线告警
- 历史数据多指标趋势可视化
- AI 智能问答辅助运维

### 1.3 技术路线
- **端侧（校内模拟）**：Mock数据发生器（Python/Flask），支持 light/temperature/humidity/power 四种传感器类型
- **端侧（基地真实）**：鸿蒙开发板(Hi3861/Hi3516) + 光照传感器 + 继电器
- **通信**：MQTT协议（EMQX Broker）
- **后端**：Java Spring Boot 3.2.5
- **前端**：Vue 3 + Element Plus + ECharts
- **数据库**：MySQL 8.0（远程服务器）

## 4. 业务流程

### 流程一：光照联动自动开关路灯 (v5)
```
1. 市政人员设置光照阈值 + 传感器策略
2. 系统实时监测多种传感器数据（MQTT 上报）
3. 后端通过 device_sensor 表解析 sensor→device
4. 根据 sensorStrategy 决策：
   ├─ single → 取主传感器 illuminance
   └─ average → 取所有 bound light 传感器平均值
5. 判断光照是否低于 thresholdOn？
   ├─ 是（且当前灯关） → SELECT FOR UPDATE 悲观锁 → 下发开灯 MQTT cmd → WebSocket 推送状态
   └─ 否 → 继续监测
6. 光照高于 thresholdOff 时自动关灯（同上逻辑）
```

### 流程二：设备离线告警处理
```
1. 传感器定期发送心跳信号（MQTT）
2. HeartbeatChecker @Scheduled 每 5s 检查心跳
3. 心跳中断超过 30s？
   ├─ 是 → 标记 offline → 触发 OFFLINE 告警 → WebSocket 推送 NEW_ALARM
   └─ 否 → 正常
4. 设备恢复上线 → 自动解除 OFFLINE 告警 → WebSocket 推送 DEVICE_STATUS
```

### 流程三：Mock 数据发生器 (v5)
```
1. 启动模拟器 → 从 SQLite (state.db) 恢复上次关闭前的传感器状态
2. 已启用的传感器自动开始上报数据
3. 上次关闭前被停止的传感器保持停止状态
4. 按传感器类型分离字段:
   light → illuminance/cloudCover/status
   temperature → temperature
   humidity → humidity
   power → voltage/power
5. 基于 Spencer 太阳赤纬 + Beer-Lambert 模型生成逼真光照曲线
```

## 5. 数据定义

### 5.1 传感器类型与字段 (v5)

| 传感器类型 | 上报字段 | 说明 |
|-----------|---------|------|
| light | illuminance, lightIntensity, cloudCover, status | 光照传感器 |
| temperature | temperature | 温度传感器 |
| humidity | humidity | 湿度传感器 |
| power | voltage, power | 电力传感器 |

### 5.2 光照强度
- **范围**：0 ~ 2000 Lux
- **阈值建议**：
  - 开灯阈值：< 50 Lux（黄昏/阴天）
  - 关灯阈值：> 100 Lux（天亮）

### 5.3 设备状态
- **online**：在线（心跳正常）
- **offline**：离线（心跳超时 > 30秒）

### 5.4 控制模式
- **auto**：自动联动（根据光照阈值自动开关灯）
- **manual**：手动控制（用户通过页面操作）

### 5.5 告警类型 (v5)

| 类型 | 触发条件 | 适用传感器 |
|------|---------|-----------|
| OFFLINE | 心跳超时 > 30s | 全部 |
| VOLTAGE_ABNORMAL | 电压 < 180V 或 > 260V | light, power |
| TEMPERATURE_HIGH | 温度 > 55°C | temperature |
| POWER_HIGH | 功率 > 200W | light, power |
| SENSOR_ABNORMAL | 数据连续 10 次未变化 | 全部 |

### 5.6 告警严重程度
- **INFO**：信息提示
- **WARNING**：警告
- **CRITICAL**：严重

### 5.7 告警状态
- **PENDING**：待处理
- **RESOLVED**：已解除（设备恢复上线时自动解除 OFFLINE 类型）

## 6. 非功能性需求

| 类别 | 要求 |
|------|------|
| 实时性 | 传感器数据延迟 < 3秒 |
| 可用性 | 后端多实例共享订阅，单点故障不影响数据接收 |
| 扩展性 | JSON 列 + JPA ddl-auto 支持新增传感器类型无需改表 |
| 安全性 | JWT 鉴权 + RBAC 权限控制 |
| 可维护性 | 前后端分离，Maven/Vite 构建标准化 |
