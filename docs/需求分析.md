# 智慧路灯系统 - 需求分析文档

## 1. 项目概述

### 1.1 项目背景
智慧路灯系统是重庆交通大学-中软国际集中实训项目之一，旨在通过「端-智-云一体化」架构，实现路灯的智能控制和节能管理。

### 1.2 项目目标
构建一套智慧路灯物联网系统，实现：
- 实时监测环境光照强度
- 根据光照阈值自动开关路灯
- 支持手动远程控制路灯
- 设备状态监控与离线告警
- 历史数据可视化展示
- 智能问答（AI辅助）

### 1.3 技术路线
- **端侧（校内模拟）**：Mock数据发生器模拟光照传感器
- **端侧（基地真实）**：鸿蒙开发板(Hi3861/Hi3516) + 光照传感器 + 继电器
- **通信**：MQTT协议
- **后端**：Java Spring Boot 3.x
- **前端**：Vue 3 + Element Plus + ECharts
- **数据库**：MySQL

## 4. 业务流程

### 流程一：光照联动自动开关路灯
```
1. 市政人员设置光照阈值
2. 系统实时监测光照强度（Mock数据/真实传感器）
3. 判断光照是否低于阈值？
   ├─ 是 → 自动下发开灯指令 → 路灯开启 → 状态反馈
   └─ 否 → 持续监测
4. 光照高于阈值时自动关灯
```

### 流程二：设备离线告警处理
```
1. 设备定期发送心跳信号（MQTT）
2. 后端监测心跳
3. 心跳中断超过阈值时间？
   ├─ 是 → 触发离线告警 → 通知管理员
   └─ 否 → 正常记录
4. 管理员处理 → 设备恢复上线
```

## 5. 数据定义

### 5.1 光照强度
- **范围**：0 ~ 2000 Lux
- **阈值建议**：
  - 开灯阈值：< 50 Lux（黄昏/阴天）
  - 关灯阈值：> 100 Lux（天亮）

### 5.2 设备状态
- **online**：在线（心跳正常）
- **offline**：离线（心跳超时 > 30秒）

### 5.3 告警类型
- **offline**：设备离线告警
- **sensor_abnormal**：传感器数据异常
