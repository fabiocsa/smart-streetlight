### 核心数据流

```
模拟器/鸿蒙设备 ──MQTT──→ EMQX Broker ──MQTT──→ Spring Boot 后端 ──WebSocket──→ Vue 3 前端
                                         ←──MQTT── (控制指令下行 → 传感器 cmd 主题)
```

1. mock-sender 或真实设备通过 MQTT 上报传感器数据，topic: `streetlight/sensor/{sensorId}/data`
2. `MqttMessageHandler` 接收 → 通过 `device_sensor` 关联表解析 sensorId → deviceId
3. `SensorDataServiceImpl.saveAndAutoControl()` 保存到 `sensor_data` 表 + 判断光照阈值自动联动
4. 实时数据通过 `WebSocketHandler.pushSensorData()` 推送到前端 `/ws/monitor`
5. 控制指令通过 MQTT 下行到传感器 cmd 主题 → 传感器响应 cmd/response

## 3. MQTT Topic 设计 (v5)

> **原则**:
> - 传感器通过 MQTT 独立注册，不携带设备 ID
> - 传感器数据使用自身 sensorId 发布；绑定关系由后端通过 device_sensor 表管理
> - 控制指令通过传感器 cmd 主题下发（不再使用设备级控制主题）
> - 后端多实例共享订阅数据主题，EMQX 轮询分发

| Topic | 方向 | Qos | 说明 |
|-------|------|-----|------|
| `streetlight/sensor/{sensorId}/data` | 传感器→后端 | 1 | 传感器数据上报（不含 deviceId） |
| `streetlight/sensor/{sensorId}/status` | 传感器→后端 | 1 | 传感器心跳/在线状态 |
| `streetlight/sensor/{sensorId}/cmd` | **后端→传感器** | 1 | 控制指令下行（开关灯、模式切换） |
| `streetlight/sensor/{sensorId}/cmd/response` | 传感器→后端 | 1 | 控制执行结果反馈 |
| `streetlight/sensor/register` | 传感器→后端 | 1 | 传感器自动注册（全局主题） |
| `streetlight/sensor/unregister` | 传感器→后端 | 1 | 传感器注销 |

**已删除的旧 topic（v3→v4）：**
- ~~`streetlight/{deviceId}/control`~~ → 改为 `streetlight/sensor/{sensorId}/cmd`
- ~~`streetlight/{deviceId}/control/response`~~ → 改为 `streetlight/sensor/{sensorId}/cmd/response`
- ~~`streetlight/mock-sender/config`~~ → 已废弃

### 数据格式 (v5 — 按传感器类型分离)

> **v5 关键变更**：每种传感器类型只发送自己的字段，不再跨类型混发。
> 模拟器 `data_generator.py` 通过 `SENSOR_TYPE_FIELDS` 字典控制每种类型的字段集。

**光照传感器 (light)：**
```json

  "sensorType": "light",
  "sensorId": 1,
  "illuminance": 125.5,
  "lightIntensity": 125.5,
  "cloudCover": 0.3,
  "status": "OFF",
  "controlMode": "auto",
  "timestamp": "2026-07-01T10:00:00Z"
}
```

**温度传感器 (temperature)：**
```json
{
  "sensorType": "temperature",
  "sensorId": 2,
  "temperature": 28.3,
  "timestamp": "2026-07-01T10:00:00Z"
}
```

**湿度传感器 (humidity)：**
```json
{
  "sensorType": "humidity",
  "sensorId": 3,
  "humidity": 65.0,
```

## 4. 数据库设计

### ER图（文字描述）

```
Device (1) ── (N) device_sensor (N) ── (1) Sensor ── (N) SensorData
  │
  │
Device (1) ────── (N) AlarmLog
  │
  │
Device (1) ────── (N) ControlLog
```

> Sensor 与 Device 是独立实体，通过 device_sensor 关联表实现 N:M 绑定。
> 传感器上报数据时可能未绑定设备，deviceId 以 sensor_{id} 作为临时标识。

## 5. 查询示例

```sql
-- 按传感器类型聚合温度小时均值
SELECT HOUR(reported_at) AS hr,
       AVG(CAST(JSON_UNQUOTE(JSON_EXTRACT(data_json, '$.temperature')) AS DOUBLE)) AS avg_temp
FROM sensor_data
WHERE sensor_type = 'temperature'
  AND reported_at >= '2026-07-01'
GROUP BY HOUR(reported_at);

-- 获取各设备最新传感器数据（按 deviceId 合并多种传感器）
SELECT sd.* FROM sensor_data sd
INNER JOIN (
  SELECT device_id, MAX(reported_at) AS max_time
  FROM sensor_data GROUP BY device_id
) latest
ON sd.device_id = latest.device_id AND sd.reported_at = latest.max_time;

-- 按指标动态映射传感器类型（趋势查询）
-- sensorTypeForMetric(): lightIntensity→light, temperature→temperature, humidity→humidity, power→power
```

## 6. 心跳与离线判断逻辑

1. HeartbeatChecker `@Scheduled(fixedRate=5000)` 每 5 秒执行
2. 扫描所有设备，比对 `lastHeartbeat` 与当前时间
3. 超过 `heartbeat.timeout`（默认30秒）→ 标记离线 → 生成 OFFLINE 告警
4. 设备恢复上线时自动解除 OFFLINE 告警
5. WebSocket 推送 DEVICE_STATUS 通知前端

## 7. 光照联动控制逻辑 (v5)

```
每收到一条传感器数据：
  1. 通过 device_sensor 表反查 deviceId
  2. 仅 sensorType="light" 时执行联动
  3. 读设备 sensorStrategy:
     - single → 取 primarySensorId 对应的传感器数据
     - average → 取所有 bound light 传感器的 illuminance 平均值
  4. SELECT ... FOR UPDATE 悲观行锁读设备（防并发重复触发）
  5. 检查 controlMode == "auto"
  6. 光照 < thresholdOn → 下发开灯指令
  7. 光照 > thresholdOff → 下发关灯指令
  8. 通过传感器 cmd 主题逐传感器发送 MQTT 控制指令
  9. 保存 ControlLog
  10. WebSocket 推送 DEVICE_STATUS
```

### 告警检测（按 sensorType 分流）

| 告警类型 | 触发条件 | 适用 sensorType |
|---------|---------|----------------|
| OFFLINE | 心跳超时 > 30s | 所有 |
| VOLTAGE_ABNORMAL | 电压 < 180 或 > 260 | light, power |
| TEMPERATURE_HIGH | 温度 > 55 | temperature |
| POWER_HIGH | 功率 > 200 | light, power |
| SENSOR_ABNORMAL | 数据连续 10 次未变化 | 所有 |

## 8. Mock 数据发生器 (v5)

`tools/mock-sender/` — 基于真实时钟 + 太阳位置模型的 Python 模拟器：

| 特性 | 说明 |
|------|------|
| 光照模型 | Spencer(1971) 太阳赤纬公式 + Beer-Lambert 大气光程 |
| 默认坐标 | 重庆: lat=29.5, lon=106.5, UTC+8 |
| Web UI | Flask (端口 5050)，卡片网格 + 模糊搜索 + 类型筛选 + 拖拽排序 |
| 持久化 | SQLite (`state.db`)，保存传感器 running/enabled/controlMode 状态 |
| 状态恢复 | 重启后自动恢复关闭前的运行状态 |
| 字段分离 | `SENSOR_TYPE_FIELDS` 字典控制每种类型只发送自己的字段 |
| 控制响应 | 通过 `streetlight/sensor/{sensorId}/cmd/response` 回传 |
