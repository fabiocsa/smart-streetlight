## 3. MQTT Topic 设计

> **原则 (v3)**:
> - 设备（路灯本体）仅通过 REST API 管理。
> - 传感器通过 MQTT 独立注册，不携带设备 ID。
> - 传感器数据使用自身 sensorId 发布；绑定关系由后端通过 device_sensor 表管理。

| Topic | 方向 | Qos | 说明 |
|-------|------|-----|------|
| `streetlight/sensor/register` | 传感器→后端 | 1 | 传感器独立注册（全局主题，无 deviceId） |
| `streetlight/sensor/{sensorId}/data` | 传感器→后端 | 1 | 传感器数据上报（使用传感器自身 ID） |
| `streetlight/sensor/{sensorId}/status` | 传感器→后端 | 1 | 传感器心跳/在线状态 |
| `streetlight/{deviceId}/control` | 后端→传感器 | 1 | 开关灯控制指令 |
| `streetlight/{deviceId}/control/response` | 传感器→后端 | 1 | 控制执行结果反馈 |
| `streetlight/mock-sender/config` | 后端→传感器 | 1 | 模拟器配置指令（含 bind/unbind 通知） |

### 设备与传感器管理 (v3)

```
设备（路灯本体）:
  1. 仅通过 REST API 管理: POST/GET/PUT/DELETE /api/devices
  2. 设备绑定传感器: POST /api/devices/{id}/bind-sensor
  3. 设备解绑传感器: POST /api/devices/{id}/unbind-sensor/{sensorId}
  4. 绑定关系存储在 device_sensor 关联表中

传感器（数据采集单元）:
  1. 独立存在，不持有设备 ID
  2. 通过 MQTT 独立注册: streetlight/sensor/register（全局主题）
  3. 通过 MQTT 发送数据: streetlight/sensor/{sensorId}/data
  4. 注册后自动进入"未绑定"状态，由管理员在设备详情页手动绑定
  5. 后端通过 device_sensor 表反查传感器所属设备
```

## 5. 传感器数据存储方案

### 设计选择：JSON 列方案

**为什么选择 JSON 列而非 EAV（实体-属性-值）？**

1. **MySQL 8.x 原生 JSON 支持**：`JSON_EXTRACT()` / `->` / `->>` 运算符可直接用于聚合查询
2. **上游数据格式对齐**：mock-sender 已输出完整 JSON payload，存储层与上游一致
3. **零 DDL 扩展**：新增传感器类型无需 `ALTER TABLE`
4. **JPA + Jackson 无缝集成**：`Map<String, Object>` ↔ JSON 自动序列化
5. **查询写法简洁**：`AVG(JSON_EXTRACT(data_json, '$.temperature'))` 与原来按列聚合差异小

### 查询示例

```sql
-- 按设备+时间范围查询光照历史
SELECT * FROM sensor_data
WHERE device_id = 'SL-001' AND reported_at BETWEEN '2026-07-01' AND '2026-07-02'
ORDER BY reported_at ASC;

-- 按传感器类型聚合温度小时均值
SELECT HOUR(reported_at) AS hr,
       AVG(CAST(JSON_UNQUOTE(JSON_EXTRACT(data_json, '$.temperature')) AS DOUBLE)) AS avg_temp
FROM sensor_data
WHERE device_id = 'SL-001' AND sensor_type = 'light'
  AND reported_at >= '2026-07-01'
GROUP BY HOUR(reported_at);

-- 获取各设备最新数据
SELECT sd.* FROM sensor_data sd
INNER JOIN (SELECT device_id, MAX(reported_at) AS max_time FROM sensor_data GROUP BY device_id) latest
ON sd.device_id = latest.device_id AND sd.reported_at = latest.max_time;
```

## 7. 光照联动控制逻辑

保持不变——仅 light 类型传感器数据触发联动：

```
每收到一条传感器数据：
  1. 提取 lightIntensity（从 data_json 中 JSON_EXTRACT）
  2. 仅 sensorType="light" 时执行联动
  3. 检查设备控制模式
  4. auto 模式 → 阈值判断 → 下发开关灯指令
  5. 保存控制日志
```
