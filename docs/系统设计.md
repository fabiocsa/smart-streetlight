## 3. MQTT Topic 设计

| Topic | 方向 | Qos | 说明 |
|-------|------|-----|------|
| `streetlight/{deviceId}/sensor/data` | 设备→后端 | 1 | 光照强度数据上报 |
| `streetlight/{deviceId}/status` | 设备→后端 | 1 | 设备心跳/在线状态 |
| `streetlight/{deviceId}/control` | 后端→设备 | 1 | 开关灯控制指令 |
| `streetlight/{deviceId}/control/response` | 设备→后端 | 1 | 控制执行结果反馈 |

### 数据格式

**传感器数据上报 (设备→后端)：**
```json
{
  "deviceId": "SL-001",
  "lightIntensity": 125.5,
  "timestamp": "2026-07-01T10:00:00Z"
}
```

**设备心跳 (设备→后端)：**
```json
{
  "deviceId": "SL-001",
  "status": "online",
  "battery": 85
}
```

**控制指令 (后端→设备)：**
```json
{
  "command": "on",
  "source": "auto",
  "timestamp": "2026-07-01T10:00:00Z"
}
```

**控制响应 (设备→后端)：**
```json
{
  "command": "on",
  "result": "success",
  "timestamp": "2026-07-01T10:00:01Z"
}
```

## 5. 心跳与离线判断逻辑

```
设备每10秒发送一次心跳（MQTT topic: status）
后端收到心跳 → 更新 device.last_heartbeat = now(), device.status = 'online'

每5秒定时检查：
  遍历所有在线设备
  如果 now() - last_heartbeat > 30秒
    → 标记 status = 'offline'
    → 创建一条离线告警（alarm_type = 'offline'）
    → WebSocket推送给前端

设备重新发送心跳
  → 标记 status = 'online'
  → 自动关闭未处理的离线告警
```

## 6. 光照联动控制逻辑

```
每收到一条传感器数据：
  1. 检查设备当前控制模式
  2. 如果是 manual 模式 → 跳过自动控制
  3. 如果是 auto 模式：
     a. 当前灯光状态为 off，光照 < threshold_on（默认50Lux）
        → 下发开灯指令
        → 记录控制日志 source=auto
     b. 当前灯光状态为 on，光照 > threshold_off（默认100Lux）
        → 下发关灯指令
        → 记录控制日志 source=auto
```
