## 业务流程

### 业务场景：光照联动自动开关路灯 (v5)

| 步骤 | 角色 | 动作 | 说明 |
|------|------|------|------|
| 1 | 操作员 | 设置光照阈值+传感器策略 | 配置 thresholdOn/Off + sensorStrategy (single/average) |
| 2 | 传感器 | 实时上报数据 | light 传感器采集 illuminance/cloudCover |
| 3 | 后端 | 通过 device_sensor 解析绑定 | sensorId → deviceId |
| 4 | 后端 | 按策略决策 | single: 取主传感器; average: 取所有 bound light 平均值 |
| 5 | 后端 | 判断阈值 | illuminance < thresholdOn 且灯关? |
| 6 | 后端 | 自动开灯 | SELECT FOR UPDATE 悲观锁 → MQTT sensor cmd 下发 on |
| 7 | 传感器 | 执行动作 | 传感器响应 cmd → 路灯开启 |
| 8 | 后端 | 状态推送 | WebSocket DEVICE_STATUS 推送到前端 |
| 9 | 后端 | 持续监测 | illuminance > thresholdOff → 自动关灯 |

### 业务场景：多类型告警处理 (v5)

| 步骤 | 角色 | 动作 | 说明 |
|------|------|------|------|
| 1 | 后端 | HeartbeatChecker 监测 | @Scheduled 每5s检查心跳 |
| 2 | 后端 | 分流检测 | 按 sensorType 检测: voltage→light/power, temperature→temperature, power→light/power |
| 3 | 后端 | 触发告警 | 生成 AlarmLog，WebSocket 推送 NEW_ALARM |
| 4 | 管理员 | 接收告警 | 告警列表查看详情 |
| 5 | 管理员 | 处理告警 | 派人现场维修/检查 |
| 6 | 设备 | 恢复上线 | 心跳恢复→自动解除 OFFLINE 告警 |

### 业务场景：Mock 模拟器传感器管理 (v5)

| 步骤 | 角色 | 动作 | 说明 |
|------|------|------|------|
| 1 | PM/开发 | 启动模拟器 | Flask Web UI 端口 5050 |
| 2 | 系统 | 从 SQLite 恢复状态 | 读取 state.db → 恢复每个传感器的 running/enabled/controlMode |
| 3 | 系统 | 自动启动已启用传感器 | enabled=True 的传感器自动开始上报 |
| 4 | 用户 | 停止不常用传感器 | 点击停止 → enabled=False, running=False 写入 SQLite |
| 5 | 用户 | 重启模拟器 | 上次停止的传感器保持停止（enabled=False） |
| 6 | 用户 | 添加/编辑/删除传感器 | 卡片式 UI 管理，支持拖拽排序 |
