## 代码管理规范

### Git分支策略
- `main`：稳定版本（答辩前合并）
- `dev`：日常开发分支
- `feature/xxx`：功能分支（如feature/mqtt、feature/dashboard）

### Commit规范
```
feat: 新功能
fix: 修复bug
docs: 文档
refactor: 重构
style: 样式
chore: 工程化
```
示例：`feat: 实现MQTT传感器数据接收`

### 代码审查
- 每人完成功能后提PR，至少1人Review后合并
- PM负责最终代码质量把关
